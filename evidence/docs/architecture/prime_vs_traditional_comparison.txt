  "gpt-4": ["reasoning", "coding"],

┌──────────────────────────────────────────────────────────────────────┐
│ Relationship Array (Rust, cache-aligned)                             │
├──────────────┬──────────────┬─────────────┬──────────┐              │
│ source       │ target       │ type        │ weight   │              │
│ (u128)       │ (u128)       │ (u32)       │ (f32)    │              │
├──────────────┼──────────────┼─────────────┼──────────┤              │
│ 0x00...1234  │ 0x00...5678  │ 2           │ 0.95     │              │
│ 0x00...1234  │ 0x00...9ABC  │ 2           │ 0.92     │              │
│ 0x00...5678  │ 0x00...DEF0  │ 3           │ 0.88     │              │
└──────────────┴──────────────┴─────────────┴──────────┘              │

Adjacency Index (HashMap):
{
  0x00...1234: [0x00...5678, 0x00...9ABC],
  0x00...5678: [0x00...DEF0, ...],
  ...
}

Size per relationship: 40 bytes (struct) + 32 bytes (index) = 72 bytes


MEMORY COMPARISON:
Traditional: 152 bytes × 300K relationships = 45.6 MB
Prime:       72 bytes × 300K relationships  = 21.6 MB
SAVINGS: 52% reduction

QUERY 1: "Find all AI entities in the Language subdomain"

TRADITIONAL (PostgreSQL + ORM):
┌─────────────────────────────────────────────────────────────────────┐
│ Step 1: SQL Query Construction                    [10-20ms]         │
│   SELECT * FROM entities                                            │
│   WHERE type = 'ai_model'                                           │
│   AND properties->>'subdomain' = 'language'                         │
│                                                                      │
│ Step 2: PostgreSQL Query Execution                [5-10ms]          │
│   - Parse SQL                                                       │
│   - Execute WHERE clauses                                           │
│   - Serialize JSONB results                                         │
│                                                                      │
│ Step 3: Network Transfer                          [5-10ms]          │
│   - PostgreSQL → Python client                                      │
│                                                                      │
│ Step 4: ORM Hydration                             [10-30ms]         │
│   - Create Entity objects                                           │
│   - Deserialize JSONB to dicts                                      │
│                                                                      │
│ TOTAL: 30-70ms                                                      │
└─────────────────────────────────────────────────────────────────────┘


PRIME-ENCODED (Rust, RAM):
┌─────────────────────────────────────────────────────────────────────┐
│ Step 1: Build Bit Mask                           [1 CPU cycle]      │
│   let mask = 0xFFFFFFFF_FFFFFFFFu128 << 64;                         │
│   let target = (AI_DOMAIN << 96) | (LANGUAGE_SUB << 64);            │
│                                                                      │
│ Step 2: SIMD Filter                               [~1000 cycles]    │
│   entries.iter()                                                    │
│     .filter(|&&addr| (addr & mask) == target)                       │
│     // AVX2: 4 addresses per cycle                                  │
│                                                                      │
│ Step 3: Collect Results                           [~500 cycles]     │
│   .collect::<Vec<u128>>()                                           │
│                                                                      │
│ TOTAL: ~0.0005ms (500 nanoseconds)                                  │
│ SPEEDUP: 60,000× - 140,000× faster!                                 │
└─────────────────────────────────────────────────────────────────────┘

PART 5: MEMORY LAYOUT COMPARISON

TRADITIONAL (150,000 entities):

Heap Memory Layout (fragmented):
┌────────┬─────┬────┬──────┬───┬─────────┬──┬────┬─────┬───────────┐
│Entity1 │...  │Ent2│ ...  │E3 │ ...     │E4│... │ ... │Entity150K │
└────────┴─────┴────┴──────┴───┴─────────┴──┴────┴─────┴───────────┘
  500B    gap   500B  gap   500B  gap     500B gap  gap    500B

- Scattered across heap (poor cache locality)
- Each entity: ~500 bytes
- Adjacency list: separate HashMap (more pointers)
- Total: ~75 MB + 30 MB (adjacency) = 105 MB

Cache Performance:
  L1 cache hit rate: ~40% (scattered data)
  L2 cache hit rate: ~60%
  L3 cache hit rate: ~80%
  DRAM access: 20% of operations (slow!)


PRIME-ENCODED (150,000 entities):

Contiguous Memory Layout:
┌──────┬──────┬──────┬──────┬──────┬───────────┬──────┐
│Addr1 │Addr2 │Addr3 │Addr4 │Addr5 │    ...    │Addr  │
│(u128)│(u128)│(u128)│(u128)│(u128)│           │150K  │
└──────┴──────┴──────┴──────┴──────┴───────────┴──────┘
  16B    16B    16B    16B    16B                 16B

- Contiguous array (excellent cache locality)
- Each address: 16 bytes
- Total: 2.4 MB (addresses only)
- Extended data (if needed): +19.2 MB
- Relationships: +21.6 MB
- Total: 43.2 MB

Cache Performance:
  L1 cache hit rate: ~95% (sequential access)
  L2 cache hit rate: ~98%
  L3 cache hit rate: ~99%
  DRAM access: 1% of operations (rare!)

SIMD Vectorization:
  AVX2: Process 4 × u128 per cycle (256-bit registers × 2)
  AVX-512: Process 8 × u128 per cycle (512-bit registers × 2)


MEMORY COMPARISON:
Traditional: 105 MB
Prime:       43 MB
SAVINGS:     59% reduction

CACHE EFFICIENCY:
Traditional: 40% L1 hit rate
Prime:       95% L1 hit rate
IMPROVEMENT: 2.4× better cache utilization
